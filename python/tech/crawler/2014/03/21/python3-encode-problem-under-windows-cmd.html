python3在windows下f.write()方法编码问题

之前由于windows下的编码问题，转用unbutu系统。 今天由于必须使用windows server运行程序，又开始尝试解决编码问题。

报错提示：

UnicodeEncodeError: 'gbk' codec can't encode character '\xa0' in position 967: 
illegal multibyte sequence

出错代码：

1 request = urllib.request.Request(urlstr)
2 fp = urllib.request.urlopen(request)
3 content = fp.read()
4 c = content.decode('utf-8')
5 f = open('test', 'w')
6 f.write(c)

报错代码为第6行

代码仅为示意

探索：

1

将

c = content.decode('utf-8')

修改为

c = content.decode('utf-8').encode('gbk')

仍然是相同的错误

失败

2

添加’ignore’参数

将

c = content.decode('utf-8')

修改为

c = content.decode('utf-8').encode('gbk', 'ignore')

报错改变，提示：

TypeError: must be str, not bytes

有进步，继续

3

由于我的目的是写入文件

尝试修改代码

f = open('test', 'w')

为

f = open('test', 'wb')

ok，成功

4

虽然成功了，但是仍然心存不解。想起w跟我说过他一般都直接读写bytes，于是新的尝试。 百度一下，才发现，原来str.encode()返回的是bytes，而bytes.decode()方法返回的是string

str.encode(encoding="utf-8", errors="strict")
Return an encoded version of the string as a bytes object. 
Default encoding is 'utf-8'. 

bytes.decode(encoding="utf-8", errors="strict")
bytearray.decode(encoding="utf-8", errors="strict")
Return a string decoded from the given bytes. 
Default encoding is 'utf-8'.

Finally 策略

不直接读写str，将所有读写内容都更改为bytes。

代码修改为：

1 request = urllib.request.Request(urlstr)
2 fp = urllib.request.urlopen(request)
3 content = fp.read() #content是bytes
4 return_content = string_process_func(content.decode('utf-8'))	
5 f = open('test', 'wb')
6 f.write(return_content.encode('utf-8'))
7 f.close()

又涨姿势鸟~

  2. python3 document bytes.decode
